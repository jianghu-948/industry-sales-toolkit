行业入门：先理解半导体行业如何运转

从设计、制造、封测、设备材料到终端应用，半导体行业的自动化机会集中在高频数据搬运、质量追溯、跨系统协同和供应链风险监测。

一、产业链图：从芯片定义到终端交付

半导体产业链是“高研发投入 + 高资本开支 + 高质量控制 + 高供应链协同”的复杂系统。

三、数据流图：从源数据到质量与经营决策

半导体数据流的关键不是“数据多”，而是批次、设备、工艺、良率、客户和财务数据可以被稳定打通。

1. 数据源层

设计源数据需求、版本、Issue、EDA 任务、IP、GDS、Tape-out Checklist。
制造源数据WIP、批次、Recipe、设备状态、SPC、缺陷、良率、OEE。
封测源数据Wafer Map、Bin、CP/FT、测试站点、封装批次、可靠性实验。
供应链源数据订单、预测、库存、交期、采购、备件、供应商资质。
客户与质量数据RMA、8D、FA、客户审计、COA、合规证明、投诉。

2. 系统与治理层

主数据与质量追溯中心
产品/料号、版本、工艺、批次、设备、客户、供应商、合同、价格、BOM。
研发设计系统PLM / ALM / EDA / Git / Issue
制造执行系统MES / SPC / APC / EAP / CMMS
封测质量系统QMS / LIMS / Test Data / FA
经营协同系统ERP / SRM / CRM / WMS / BI
数据治理要求唯一编码、版本控制、批次追溯、权限分级、审计日志、接口监控。

3. 应用输出层

质量与良率预警良率波动、SPC 异常、设备瓶颈、异常批次、FA 聚类。
客户交付输出COA、8D、FA 报告、测试摘要、批次追溯、合规证明。
经营输出订单履约、库存周转、成本毛利、产能利用率、回款预测。
供应链输出缺料预警、供应商绩效、交期风险、备件消耗、采购对账。
内控与合规输出出口管制、客户审计、资质过期、越权访问、流程逾期。

四、名词释义：销售沟通必懂词汇

按业务模型、系统名词和制造术语组织，帮助销售快速对齐客户语境。

业务模型

Fabless无晶圆厂设计公司，专注芯片设计，制造委托 Foundry。
Foundry晶圆代工厂，为客户提供晶圆制造服务。
IDM集设计、制造、封测或部分环节于一体的综合器件厂商。
OSAT外包封装测试厂，承接芯片封装与测试服务。
EDA/IP设计工具和可复用功能模块，是芯片设计基础设施。

系统名词

MES制造执行系统，管理 WIP、工单、站点、批次和制造流程。
SPC统计过程控制，用于监测工艺稳定性和异常趋势。
APC/EAP先进过程控制/设备自动化程序，连接设备与制造系统。
QMS质量管理系统，管理异常、CAPA、8D、客户投诉和审计。
PLM产品生命周期管理，管理产品版本、BOM、变更和项目资料。

制造术语

WIP在制品，表示工厂中正在流转的晶圆或批次。
Recipe设备工艺参数配方，是制造过程稳定性的关键对象。
Yield良率，衡量可用芯片占比，是制造企业核心指标。
CP/FT晶圆测试/成品测试，是封测环节的关键数据来源。
8D/FA问题解决报告/失效分析，常用于客户质量闭环。

研发运营 / PMO

典型痛点项目周报、Issue、里程碑、客户交付资料靠人工整理。

典型痛点EDA License、服务器作业、权限申请和日志巡检分散。
RPA 场景定时抓取 License 使用率、作业排队状态和权限到期清单，生成资源使用报表。

财务 / 采购

典型痛点流片费用、IP 授权、EDA 账单、外协结算复杂。
RPA 场景下载流片/IP/EDA账单，匹配合同、项目编号和成本中心，输出对账差异清单。

最佳销售策略

先从项目资料自动汇总、流片费用对账、EDA 资源监控切入。

RPA机会点

✓项目多、版本多、人工汇总耗时。
✓研发系统不宜大改，RPA 低侵入优势明显。
✓费用、合同、外协数据适合自动核对。

制造运营

典型痛点MES、SPC、设备系统数据分散，人工导出合并。
RPA 场景从 MES/SPC/设备系统导出 WIP、停机、异常批次数据，按模板生成日报和待跟进清单。

质量 / 良率

典型痛点SPC 异常、良率波动、客户质量报告依赖人工整理。
RPA 场景抓取良率、SPC超限记录和客户投诉资料，生成质量日报、异常清单和8D/FA资料包。

供应链 / 财务

典型痛点材料、备件、设备服务采购和对账工作量大。
RPA 场景下载供应商账单并匹配PO/收货/发票，生成备件低库存清单和采购差异清单。

最佳销售策略

从“制造日报自动生成 + 质量异常清单 + 供应商对账差异”组合切入。

RPA机会点

✓系统多、数据多、跨部门协同重。
✓良率和质量问题具有强管理价值。
✓供应链金额大，对账 ROI 直观。

✓供应链波动带来高频查询和催办。
✓供应商质量资料有明确规则。
✓采购财务对账 ROI 容易量化。

一眼判断：半导体行业 RPA 机会在哪里

销售不用先判断技术边界，先识别这些高概率商机信号。

销售识别雷达
高频报表

每天/每周固定导出 WIP、良率、订单、库存、项目状态。

多系统割裂

MES、SPC、ERP、QMS、CRM、Excel、邮件之间来回切换。

质量追溯

涉及批次、设备、工艺参数、测试记录、客户投诉和 8D/FA资料归集。

供应链波动

材料、备件、交期、供应商资质和对账需要持续查询、更新和催办。

ROI 易量化

能算出节省工时、缩短周期、减少差错和降低逾期。

优先找谁

Foundry/IDM、Fabless、OSAT、设备厂商、材料厂商。

优先找什么部门

制造运营、质量良率、供应链、财务、研发运营、售后。

优先聊什么价值

减少人工、稳定留痕、缩短处理周期、提前发现风险。

制造与良率：晶圆厂最优先的高价值切入口

关键词：MES、SPC、WIP、OEE、良率、异常批。

适用客户

高频跨系统ROI 清晰

良率预警管理可见
销售话术“RPA 不做工艺判断，只负责把异常线索按规则汇总出来，让工程师更快定位。”

从MES/SPC/QMS目录归集批次履历、设备、Recipe、测试结果和质量记录，生成资料包目录。

追溯质量低侵入
销售话术“异常批处理最耗时的是找资料，RPA 可以先把资料目录和缺失项清单准备好。”

研发与流片：项目资料、版本和费用管理适合轻量切入

销售打法：重点是“会识别、会开口、会推进”

Sales Playbook

建议这样聊

  • “哪些报表每天/每周都要人工导出和合并？”
  • “哪些流程涉及批次、设备、测试或客户质量追溯？”
  • “哪些账单、订单、发票月底最容易加班？”
  • “哪些系统之间现在还靠 Excel 或邮件搬数据？”

不要一上来这样聊

  • 不要先讲平台能力和技术架构。
  • 不要先否定客户 MES、SPC、ERP 等现有系统。
  • 不要承诺替代工程判断和质量判断。
  • 不要一开始就讲全厂级大方案。
首轮拜访目标找到 1 个高频重复、跨系统、能算 ROI 的小场景。
最佳切入顺序供应链/财务对账 → 制造报表 → 质量追溯 → 研发流片 → 售后服务。
最容易成交的价值表达节省工时、降低差错、缩短周期、增强批次追溯。
销售要带回的信息系统名称、数据来源、人工步骤、频次、参与人数、当前耗时、差错影响。
半导体行业卖 RPA，不是卖“机器人”，而是卖“跨系统数据稳定流转 + 质量追溯提效 + 供应链风险提前发现”。
